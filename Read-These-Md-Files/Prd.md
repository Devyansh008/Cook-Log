# Product Requirements Document (PRD) - CookLog

## 1. Objective & Purpose
CookLog is a specialized public notebook application designed for developers to showcase their deep-dive problem-solving capabilities to recruiters. It highlights self-taught concepts, textbook problems, and advanced topics (e.g., algorithmic or mathematical challenges) that standard certificates and platforms do not capture. It is where a developer logs what they are "cooking" outside of standard coursework.

## 2. Core MVP Features
* **User Authentication:** Simple local or mock session state (scalable to third-party providers like Google/GitHub).
* **Book & Category Management:** Ability to group questions under specific textbooks, topics, or domains.
* **Q&A Logging Interface:** A private form to input question titles, problem statements, and comprehensive solutions using code and Markdown.
* **Public Recruiter Profile:** A clean, highly optimized read-only layout showing logged questions, sortable by category and topic.

## 3. Scope Exclusions (Post-MVP)
* Public user comments or community upvoting.
* Multi-user global search (focus is strictly on individual portfolio presentation).
* Advanced real-time code execution environments.
